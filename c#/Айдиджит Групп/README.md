**Задание:**

Создать Web-приложение, содержащее единственную страницу. В верхней части страницы поле для ввода строки для фильтрации. Ниже - список элементов. Каждый элемент - это заголовок и текст под ним. При каждом изменении значения фильтра остаются видимыми только элементы, содержащие введенный текст (либо в заголовке, либо в тексте). Если фильтр пустой - показываются все. Элементы загружаются с сервера. Список элементов на сервере фиксированный, база данных не нужна.

**Требования:**

net6.0, кроссплатформенность.

Использовать ASP.NET Core Web API, рендеринг на стороне клиента.
Angular 13+ или чистый JS или jQuery, другие фреймворки не использовать.

Приложение должно быть написано с учетом best practices написания кода, архитектура должна изначально быть рассчитана на расширение функциональности (переход к БД, возможность редактирования списка и пр.).

Unit tests не писать.

**Используемый стек**
.net core 6.0 web api (+ swagger) - "чистая архитектура", jquery 3.6.0, bootstrap 4

Размещение 

~ Для предупреждения ошибки cors в браузере, необходимо запустить Chrome с отключенным CORS или разместить на сервере (был использован Apache 2.4)
