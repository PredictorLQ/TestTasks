# Техническое Задание

## Проект: Система отчетности на базе Stimulsoft

**Тип решения:** Web-приложение

## 1. Общее описание

**Цель проекта:** Разработка веб-приложения для управления, генерации и планирования отчетов на основе конструктора Stimulsoft Reports, с поддержкой подключения к различным источникам данных.

## 2. Основные возможности

### 2.1. Генерация отчетов

- Импорт и использование отчетов, созданных в Stimulsoft Designer.
- Генерация отчетов в онлайн-режиме через веб-интерфейс.

### 2.2. Планировщик отчетов (Scheduled Reports)

- Возможность настройки расписания выполнения отчетов.
- Указание параметров для каждого задания:
  - Выбор источника данных
  - Выбор отчета для формирования
- Отображение готовых отчетов в приложении.

## 3. Поддерживаемые базы данных

- PostgreSQL
- Доступ в приложении с использованием Dapper

## 4. Аутентификация и управление пользователями

### 4.1. Аутентификация

- Использование Keycloak для авторизации по протоколу OIDC (OpenID Connect).
- Пользователь проходит аутентификацию через Keycloak и получает доступ к системе.

### 4.2. Управление пользователями

- После первого входа пользователя через Keycloak его данные сохраняются в локальном списке пользователей системы.

## 5. Интеграция и настройки

- Централизованное хранение конфигурации логгирования и подключения к бд.
- Возможность добавления, редактирования и удаления источников данных.
- Использовать Docker Compose для интеграции компонентов приложения

## 6. Требования к интерфейсу

- Веб-интерфейс для:
  - Авторизации пользователей;
  - Загрузки, выбора и запуска отчетов;
  - Управления заданиями планировщика;
- Использовать Tailwind CSS
